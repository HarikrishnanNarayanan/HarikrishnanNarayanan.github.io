Understanding Natural Language Interpretation¶

Semantic relationships between sentences are applied in various fields like in machine translation and sentiment analysis. As the same sentence can be said in a different way with a lot of alterations, detecting duplicate questions or finding the similarity between sentences is a difficult task. This process involves understanding the dataset, preprocessing the data and training different models to classify the pair of sentences as duplicate or not duplicate sentences[1]. Duplicate questions and duplicate responses are present in almost all the online forums. This is a problem as it stops a person from seeing the best response to the query or the comment that he/she has made[2]. Predicting whether a question is a duplicate or not can help the forums to automatically redirect the user queries to the already present query and response, and also help to prevent users from posting responses with a similar content which is already present.

Machine learning and deep learning algorithms are now widely used for NLP tasks. In this project, we analyzed different methods to extract the semantic relationship between pairs of sentences and classified them as duplicate or not duplicate. Our objective is to find the best solution that works well in classifying the duplicate sentences for any dataset that contains a pair of sentences. We explore various traditional NLP algorithms, Machine learning approaches, Deep learning methods, and Ensemble methods to identify duplicate sentences and questions. The datasets that we used for this analysis are Quora question pair data and MSR Paraphrase Corpus. Identifying whether a pair of questions are duplicate or not is treated as a binary classification problem.

Purpose of the study are:

  • To find the best combination of feature extraction techniques and Machine Learning algorithm to detect duplicate with high accuracy and F1 score
  • Finding the optimal parameters for the methods so that it can be applied to multiple data sets
  • Comparing the performance of the identified solution on multiple datasets and finding the best model that performs well on both the data set

df = pd.read_csv('Data/train.csv')

seed = 123

#Creating the embedding matrix
NUM_WORDS = len(embeddings) #200000
tokenize = Tokenizer(num_words = NUM_WORDS)
tokenize.fit_on_texts(q)
word_index = tokenize.word_index

q1_train = tokenize.texts_to_sequences(train['question1'])
q2_train = tokenize.texts_to_sequences(train['question2'])
q1_test = tokenize.texts_to_sequences(test['question1'])
q2_test = tokenize.texts_to_sequences(test['question2'])

max_seq_length = max_seq_len(q1_train)

max_seq_length = max_seq_len(q1_test, max_seq_length)
max_seq_length = max_seq_len(q2_test, max_seq_length)

q1_train_padded = pad_sequences(q1_train, max_seq_length)
q2_train_padded = pad_sequences(q2_train, max_seq_length)
q1_test_padded = pad_sequences(q1_test, max_seq_length)
q2_test_padded = pad_sequences(q2_test, max_seq_length)

#Matrix with the embedding weights
embedding_dim = 300
embedding_weights = create_embedding_weights(embeddings, embedding_dim, word_index, NUM_WORDS)
